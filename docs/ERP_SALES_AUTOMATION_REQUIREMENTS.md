# ERP 매출등록 자동화 요구사항

## 목적

WPF 프로그램에서 수량, 단가, 거래처코드, 계정코드를 입력받고, 이미 로그인 정보가 저장된 Chrome 세션을 이용해 ERP 매출등록과 전기 절차를 자동 수행한다.

## 입력값

- 수량
- 단가
- 거래처코드
- 계정코드

## 자동화 절차

1. Chrome 연결을 확인한다.
2. ERP 로그인 페이지에 접속한다.
   - `https://ibcenter.co.kr/erp/erp/erplogin/erplogin_dispatch.jsp`
3. 아이디와 비밀번호는 Chrome에 이미 저장되어 있으므로 입력값은 건드리지 않는다.
4. 로그인 버튼만 클릭한다.
5. 로그인 성공 여부를 확인한다.
6. 로그인에 성공하면 로그인된 탭은 닫지 말고 그대로 둔다.
7. 회계관리 버튼을 클릭한다.
8. 거래전표 메뉴를 펼친다.
9. 거래전표(매출등록) 메뉴를 펼친다.
10. 원화 버튼을 클릭한다.
11. 거래일자에 오늘 날짜를 입력한다.
12. 차변 드롭다운 버튼을 눌러 `외상매출금 [1141]`을 선택한다.
13. 매출구분 드롭다운 버튼을 눌러 `서비스(사회및개인)업 폐차처리업`을 선택한다.
14. 거래처코드/명 칸에 입력받은 거래처코드를 입력하고 Enter를 실행한다.
15. 담당부서 칸에 `20`을 입력하고 Enter를 실행한다.
16. 전자(세금)계산서 발송구분 드롭다운 버튼을 누르고 `국세청HTS`를 클릭한다.
17. 화면 하단 라인 입력 영역에서 품목코드/품목명(적요) 라벨 오른쪽의 긴 텍스트 입력칸에 `차피 압축`을 입력한다.
18. 수량/단가 영역에서 수량 칸에 입력받은 수량을 입력한다.
19. 단가 칸에 입력받은 단가를 입력한다.
20. 계산 버튼을 클릭한다.
21. 계산 결과가 정상 반영되었는지 확인한다.
    - 예: 수량 `15,000`, 단가 `275`
    - 공급가액 `4,125,000`
    - 세액 `412,500`
22. 계정코드(대변) 칸에 입력받은 계정코드를 입력하고 Enter를 실행한다.
23. 화면 하단 라인 입력 영역에서 `라인저장(L)` 버튼을 클릭한다.
24. 저장 후 라인 목록에 입력한 품목, 수량, 단가, 공급가액, 세액이 반영되었는지 확인한다.
25. 상단의 `거래전기[S]` 버튼을 클릭한다.
26. 화면이 전기 완료 상태로 바뀌었는지 확인한다.
27. 상단의 `회계전표 동일자생성` 버튼을 클릭한다.
28. 회계전표입력 화면으로 이동했는지 확인한다.
29. 상단의 `원장전기[P]` 버튼을 클릭한다.
30. `원장전기: 완료` 상태가 표시되는지 확인한다.

## 주의사항

- 아이디나 비밀번호를 새로 입력하지 않는다.
- 저장된 로그인 정보가 자동 입력되어 있으면 그대로 사용한다.
- 비밀번호 입력이나 변경을 요청하지 않는다.
- 클릭이 반영되지 않으면 현재 화면에 보이는 버튼 위치를 기준으로 직접 클릭한다.
- 계산 버튼 클릭 후 공급가액/세액이 `0`이면 수량과 단가를 다시 입력하고 계산 버튼을 다시 클릭한다.
- `라인저장(L)`은 반드시 저장 전 입력값과 계산 결과를 확인한 뒤 클릭한다.
- 각 단계 완료 후 현재 탭은 닫지 말고 그대로 열어둔다.

## 구현 방향

- WPF + MVVM 패턴으로 작성한다.
- MVVM 보일러플레이트는 `CommunityToolkit.Mvvm`을 사용한다.
- Chrome은 원격 디버깅 포트 `9222`에 연결한다.
- ERP 로그인 아이디/비밀번호 입력 필드는 자동화 코드에서 직접 수정하지 않는다.
- 자동화 진행 상황과 실패 지점은 WPF 화면 로그에 남긴다.
- 기능 추가 단위로 커밋하고 원격 저장소에 푸시한다.
