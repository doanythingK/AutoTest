# ERP 매출등록 자동화 검수표

이 문서는 요구사항 30단계가 WPF 프로그램과 문서에서 어디에 반영되어 있는지 확인하기 위한 검수표이다.

## 검수 기준

- 구현 위치: `src/AutoTest.ErpAutomation/Services/ErpAutomationService.cs`, `src/AutoTest.ErpAutomation/ViewModels/MainWindowViewModel.cs`
- 운영 문서: `README.md`, `docs/RUNBOOK.md`, `docs/IMPLEMENTATION_NOTES.md`
- 실제 완료 판정: ERP 화면에서 자동화 실행 후 실행 로그와 화면 결과를 함께 확인해야 한다.
- 현재 이 문서는 코드/문서 정합성 검수표이며, 실제 ERP 운영 검증을 대체하지 않는다.

## 입력값

| 항목 | 화면 반영 | 구현 위치 |
| --- | --- | --- |
| 수량 | WPF 입력값 | `MainWindowViewModel.QuantityText`, `AutomationInput.Quantity` |
| 단가 | WPF 입력값 | `MainWindowViewModel.UnitPriceText`, `AutomationInput.UnitPrice` |
| 거래처코드 | WPF 입력값 | `MainWindowViewModel.ClientCode`, `AutomationInput.ClientCode` |
| 계정코드 | WPF 입력값 | `MainWindowViewModel.CreditAccountCode`, `AutomationInput.CreditAccountCode` |

## 자동화 단계 대응

| 단계 | 요구사항 | 구현/검증 방식 |
| --- | --- | --- |
| 1 | Chrome 연결 확인 | `RunAutomationAsync`에서 `ChromeConnectionService.CheckConnectionAsync` 호출 |
| 2 | ERP 로그인 페이지 접속 | `ErpAutomationService.RunAsync`에서 로그인 URL로 이동 |
| 3 | 아이디/비밀번호 미입력 | 로그인 필드 입력 코드 없음, credential 입력칸 후보 제외 |
| 4 | 로그인 버튼만 클릭 | 로그인 페이지 안정 대기 후 `로그인` 텍스트 클릭 |
| 5 | 로그인 성공 확인 | `회계관리`, `로그아웃` 텍스트 확인 |
| 6 | 로그인 탭 유지 | 자동화 중 탭 닫기 호출 없음 |
| 7 | 회계관리 클릭 | `회계관리` 텍스트 클릭 |
| 8 | 거래전표 메뉴 펼침 | `거래전표` 텍스트 클릭 |
| 9 | 거래전표(매출등록) 펼침 | `거래전표(매출등록)` 텍스트 클릭 |
| 10 | 원화 클릭 | `원화` 텍스트 클릭 |
| 11 | 오늘 날짜 입력 | 실행 직전 오늘 날짜를 `yyyy-MM-dd`, `yyyy.MM.dd`, `yyyyMMdd` 후보로 입력 |
| 12 | 차변 외상매출금 [1141] 선택 | `차변` 라벨 주변 드롭다운에서 `외상매출금 [1141]`, `외상매출금` 순서로 선택 |
| 13 | 매출구분 선택 | `매출구분` 라벨 주변 드롭다운에서 `서비스(사회및개인)업 폐차처리업` 선택 |
| 14 | 거래처코드/명 입력 후 Enter | 입력값 입력, Enter, ERP 처리 표시 대기 |
| 15 | 담당부서 20 입력 후 Enter | `20` 입력, Enter, ERP 처리 표시 대기 |
| 16 | 전자세금계산서 발송구분 국세청HTS | `전자(세금)계산서 발송구분` 라벨 주변 드롭다운에서 `국세청HTS` 선택 |
| 17 | 품목명 차피 압축 입력 | 하단 영역 우선, 긴 입력칸 우선으로 `차피 압축` 입력 |
| 18 | 수량 입력 | 하단 영역 우선으로 WPF 수량 입력값 입력, 숫자 전용 입력칸이면 콤마 없는 값 사용 |
| 19 | 단가 입력 | 하단 영역 우선으로 WPF 단가 입력값 입력, 숫자 전용 입력칸이면 콤마 없는 값 사용 |
| 20 | 계산 클릭 | `계산` 텍스트 클릭 |
| 21 | 계산 결과 확인 | 공급가액/세액 라벨 주변 기대값 확인, 0 또는 미반영 시 수량/단가 재입력 후 재계산 |
| 22 | 계정코드(대변) 입력 후 Enter | 하단 영역 우선으로 계정코드 입력, Enter, ERP 처리 표시 대기 |
| 23 | 라인저장 전 확인 후 클릭 | 품목, 수량, 단가, 공급가액, 세액 확인 후 `라인저장` 하단 후보 클릭 |
| 24 | 라인 목록 반영 확인 | grid/row 후보에서 품목, 수량, 단가, 공급가액, 세액이 같은 행에 있는지 확인 |
| 25 | 거래전기 클릭 | `거래전기` 상단 후보 클릭 |
| 26 | 전기 완료 확인 | `거래전기 완료`, `거래전기: 완료`, `전기 완료` 계열 텍스트 확인 |
| 27 | 회계전표 동일자생성 클릭 | `회계전표 동일자생성` 상단 후보 클릭 |
| 28 | 회계전표입력 화면 이동 확인 | 기존 탭 또는 새 탭에서 `회계전표입력` 확인 |
| 29 | 원장전기 클릭 | `원장전기` 상단 후보 클릭 |
| 30 | 원장전기 완료 확인 | `원장전기: 완료`, `원장전기 완료` 텍스트 확인 |

## 운영 검수 항목

- Chrome 원격 디버깅 포트 연결이 성공하는지 확인한다.
- 저장된 ERP 아이디/비밀번호가 자동 입력되는지 확인한다.
- 자동화 실행 중 아이디/비밀번호 입력칸이 변경되지 않는지 확인한다.
- 수량 `15,000`, 단가 `275` 기준 공급가액 `4,125,000`, 세액 `412,500`이 표시되는지 확인한다.
- 라인저장 후 목록 행에 품목, 수량, 단가, 공급가액, 세액이 같은 행으로 표시되는지 확인한다.
- 거래전기 완료, 회계전표입력 이동, 원장전기 완료 상태가 실행 로그와 ERP 화면에 남는지 확인한다.
- 실패 시 `%LOCALAPPDATA%\AutoTest.ErpAutomation\Failures`에 PNG 또는 HTML 자료가 저장되는지 확인한다.
