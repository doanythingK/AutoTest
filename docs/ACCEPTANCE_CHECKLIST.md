# ERP 매출등록 자동화 검수표

이 문서는 요구사항 30단계가 WPF 프로그램과 문서에서 어디에 반영되어 있는지 확인하기 위한 검수표이다.

## 검수 기준

- 구현 위치: `src/AutoTest.ErpAutomation/Services/ErpAutomationService.cs`, `src/AutoTest.ErpAutomation/ViewModels/MainWindowViewModel.cs`
- 운영 문서: `README.md`, `docs/RUNBOOK.md`, `docs/IMPLEMENTATION_NOTES.md`
- 실제 완료 판정: ERP 화면에서 자동화 실행 후 실행 로그와 화면 결과를 함께 확인해야 한다.
- 현재 이 문서는 코드/문서 정합성 검수표이며, 실제 ERP 운영 검증을 대체하지 않는다.

## 입력값

| 항목 | 화면 반영 | 구현 위치 |
| --- | --- | --- |
| 수량 | WPF 입력값 | `MainWindowViewModel.QuantityText`, `AutomationInput.Quantity` |
| 단가 | WPF 입력값 | `MainWindowViewModel.UnitPriceText`, `AutomationInput.UnitPrice` |
| 거래처코드 | WPF 입력값 | `MainWindowViewModel.ClientCode`, `AutomationInput.ClientCode` |
| 계정코드 | WPF 입력값 | `MainWindowViewModel.CreditAccountCode`, `AutomationInput.CreditAccountCode` |

거래일자는 사용자 입력값이 아니다. WPF에는 실행일 기준 오늘 날짜를 표시만 하고, `AutomationInput.TryParse`는 수량, 단가, 거래처코드, 계정코드 4개만 받아 오늘 날짜를 내부에서 생성한다.

## 자동화 단계 대응

| 단계 | 요구사항 | 구현/검증 방식 |
| --- | --- | --- |
| 1 | Chrome 연결 확인 | `RunAutomationAsync`에서 `ChromeConnectionService.CheckConnectionAsync` 호출 |
| 2 | ERP 로그인 페이지 접속 | `ErpAutomationService.RunAsync`에서 로그인 URL로 이동 |
| 3 | 아이디/비밀번호 미입력 | 로그인 필드 입력 코드 없음, credential 입력칸 후보 제외 |
| 4 | 로그인 버튼만 클릭 | 로그인 페이지 안정 대기 후 보이는 로그인 관련 입력칸 전체의 저장 정보 자동 채움을 확인하고 `로그인` 텍스트 클릭. 자동 채움이 없으면 아이디/비밀번호 입력 없이 중단 |
| 5 | 로그인 성공 확인 | `회계관리`, `로그아웃` 텍스트 확인 |
| 6 | 로그인 탭 유지 | 자동화 중 탭 닫기 호출 없음 |
| 7 | 회계관리 클릭 | `회계관리` 텍스트 클릭 |
| 8 | 거래전표 메뉴 펼침 | `거래전표` 텍스트 클릭 |
| 9 | 거래전표(매출등록) 펼침 | `거래전표(매출등록)` 텍스트를 먼저 찾고, 화면에 `매출등록`만 보이면 대체 텍스트로 클릭 |
| 10 | 원화 클릭 | `원화` 텍스트 클릭 |
| 11 | 오늘 날짜 입력 | 실행 직전 오늘 날짜를 `yyyy-MM-dd`, `yyyy.MM.dd`, `yyyyMMdd` 후보로 입력하고 거래일자 라벨 주변 반영 확인 |
| 12 | 차변 외상매출금 [1141] 선택 | `차변`, `차변계정` 라벨 주변 드롭다운에서 `외상매출금 [1141]`, `외상매출금` 순서로 선택. 코드/이름 순서가 바뀌어도 단어/숫자 토큰으로 재비교하고 선택 후 라벨 주변에서 `외상매출금` 또는 `1141` 반영 확인 |
| 13 | 매출구분 선택 | `매출구분`, `매출 구분`, `구분` 라벨 주변 드롭다운에서 `서비스(사회및개인)업 폐차처리업` 또는 축약 표기 선택 후 `매출구분`, `매출 구분` 라벨 주변에서 `폐차처리업` 반영 확인 |
| 14 | 거래처코드/명 입력 후 Enter | 입력값 입력 직후 거래처코드/명 라벨 주변 반영 확인, Enter, ERP 처리 표시 대기 |
| 15 | 담당부서 20 입력 후 Enter | `20` 입력 직후 담당부서 라벨 주변 반영 확인, Enter, ERP 처리 표시 대기 |
| 16 | 전자세금계산서 발송구분 국세청HTS | `전자(세금)계산서 발송구분`, `전자세금계산서 발송구분`, `발송구분` 라벨 주변 드롭다운에서 `국세청HTS` 또는 축약 표기 선택 후 라벨 주변에서 `국세청` 또는 `HTS` 반영 확인 |
| 17 | 품목명 차피 압축 입력 | 하단 영역 우선, 긴 입력칸 우선으로 `차피 압축` 입력 후 품목명 라벨 주변 반영 확인 |
| 18 | 수량 입력 | 하단 영역 우선으로 WPF 수량 입력값 입력, 숫자 전용 입력칸이면 콤마 없는 값 사용 |
| 19 | 단가 입력 | 하단 영역 우선으로 WPF 단가 입력값 입력, 숫자 전용 입력칸이면 콤마 없는 값 사용 |
| 20 | 계산 클릭 | `계산` 텍스트 클릭 |
| 21 | 계산 결과 확인 | 공급가액/세액 라벨 주변 기대값 확인, 0 또는 미반영 시 수량/단가 재입력 후 재계산 |
| 22 | 계정코드(대변) 입력 후 Enter | 하단 영역 우선으로 계정코드 입력 직후 대변 계정 라벨 주변 반영 확인, Enter, ERP 처리 표시 대기 |
| 23 | 라인저장 전 확인 후 클릭 | 품목명 존재 여부와 수량, 단가, 공급가액, 세액 라벨 주변 기대값 확인 후 `라인저장` 하단 후보 클릭 |
| 24 | 라인 목록 반영 확인 | grid/row 후보 또는 화면상 같은 가로줄의 셀 유사 요소에서 품목, 수량, 단가, 공급가액, 세액이 같은 행에 있는지 확인. 미반영 시 저장 전 값들을 재확인하고 `라인저장`을 한 번만 재클릭 |
| 25 | 거래전기 클릭 | `거래전기` 상단 후보 클릭 |
| 26 | 전기 완료 확인 | `거래전기 완료`, `거래전기: 완료`, `전기 완료` 계열 텍스트 확인. 단계 대기 시간 안에 보이지 않으면 `거래전기`를 한 번 더 클릭하고 재확인 |
| 27 | 회계전표 동일자생성 클릭 | `회계전표 동일자생성` 상단 후보 클릭 |
| 28 | 회계전표입력 화면 이동 확인 | 기존 탭 또는 새 탭에서 `회계전표입력` 확인 |
| 29 | 원장전기 클릭 | `원장전기` 상단 후보 클릭 |
| 30 | 원장전기 완료 확인 | `원장전기: 완료`, `원장전기 완료` 텍스트 확인. 단계 대기 시간 안에 보이지 않으면 `원장전기`를 한 번 더 클릭하고 재확인 |

## 운영 검수 항목

- Chrome 원격 디버깅 포트 연결이 성공하는지 확인한다.
- 저장된 ERP 아이디/비밀번호가 자동 입력되는지 확인한다.
- 자동화 실행 중 아이디/비밀번호 입력칸이 변경되지 않는지 확인한다.
- 수량 `15,000`, 단가 `275` 기준 공급가액 `4,125,000`, 세액 `412,500`이 표시되는지 확인한다.
- 라인저장 후 목록 행에 품목, 수량, 단가, 공급가액, 세액이 같은 행으로 표시되는지 확인한다.
- 거래전기 완료, 회계전표입력 이동, 원장전기 완료 상태가 실행 로그와 ERP 화면에 남는지 확인한다.
- 실패 시 `%LOCALAPPDATA%\AutoTest.ErpAutomation\Failures`에 PNG 또는 HTML 자료가 저장되는지 확인한다.
- 실패 HTML에 마지막 자동화 단계, 오류 메시지, 앱 버전, 소스 커밋, 프레임별 보이는 버튼, 입력칸, 드롭다운 후보와 좌표 요약이 저장되고, 로그인 정보로 보이는 입력값은 실패 PNG, 요약, 전체 프레임 HTML에서 마스킹되는지 확인한다.
- 실행 종료 후 `%LOCALAPPDATA%\AutoTest.ErpAutomation\VerificationReports`에 30단계 실행 대조 Markdown 리포트가 저장되는지 확인한다.
- WPF의 `검증 리포트 폴더`, `최근 검증 리포트` 버튼으로 검증 리포트를 열 수 있는지 확인한다.
- 자동화 종료 시 최근 실행 로그, 검증 리포트, 실패 자료 zip이 만들어지고, WPF의 `검증 자료 압축` 버튼으로 다시 만들 수 있는지 확인한다.
- 실패 자료 공유 전 ERP 아이디, 비밀번호, 개인 인증 정보가 보이지 않는지 확인하고, 거래처코드/계정코드/금액 등 민감한 업무값은 필요한 범위만 남긴다.
